Web - How does it work?

Whenever you access a website, that means there is a server somewhere that is giving your browser the necessary information to display the website. Confused?

Well, a server is nothing more than a computer that is running (ideally) 100% of the time. That server is running specific code to serve your browser the files and information it needs to show you a website.

A browser is something a lot easier to understand, since you’re most likely using one at this very moment to read this article! They take HTML, CSS and JavaScript (more on these later) and show you the resulting website. Here are some of the most popular browsers: Google Chrome, Firefox, Internet Explorer, Safari, Opera, etc. Some browsers are a bit smarter than others which means it is easier to develop to, say, Google Chrome’s latest version than Internet Explorer 6.
I strongly suggest using Google Chrome to follow along these learning series, as that’s what I’ll be using as well.

Some very smart people figured out you can use these Web technologies in pretty much any platform, either by emulating the browser’s behavior or by building a wrapper, which is something like having an app that is in fact a browser window that opens your Web files.
This resulted in the fact that Web Development can currently be used for pretty much anything in any platform. How cool is that?

Alright, so how does one develop for the Web?

Web Development is comprised of:

  • Back-end - The server runs specific code and serves the necessary files and information;
  • Front-end - What the end-user will see and interact with;

The Back-end is the code that runs on the server. Usually this means serving the necessary files and information to the Front-end. Sensitive code, like for example handling authentication or accessing a database directly, will also happen here. It doesn’t really matter what programming language is used here, so you’re free to use the one you prefer. The Back-end may also be known as the API (Application Programming Interface) in Web Development. This means it has a set of endpoints that we, as developers, can access to retrieve the necessary information for our Front-end.

The Front-end is the code that will run on the user’s browser. This means things like the structure of the current page, how it looks and how it behaves. The Front-end can make requests to the Back-end whenever it needs, either to get more information or run some process on the server, for example. We’ll talk more about the Front-end in a bit.

Some developers specialize in either the Back-end or the Front-end. As a Full-Stack Developer, I work on every single step of the process, which obviously includes these two main layers.
Most modern Web Applications are extremely focused on the Front-end, with the Back-end usually being a simple API that will helps us access our data and do some other sensitive work that we may need in our app.

So what is the Front-end comprised of?

  • HTML (HyperText Markup Language) - The structure of a page;
  • CSS (Cascading Style Sheets) - The look and style of a page;
  • JS (JavaScript) - The functionality of a page;

This is the big Triforce of Web Development. You may have heard of other frameworks and/or languages but they all come down to this in the end.

Before starting a project, let’s talk a bit about HTML, CSS and JS.

HTML

HTML is the language that defines the structure of our pages. It is composed of several tags.

<a href="https://www.google.com">Click here to go to Google!</a>

This is an anchor tag, for example. It tells the browser to show a link to Google’s website with the text Click here to go to Google!

Tags usually start with <tag> and end with </tag> where tag is the tag type we want to use. In this case, we used a for anchor and everything between the start and end tags will behave as a link. Whenever you find something inside a tag, like href in this case, it is called an attribute. An attribute is like a property of the tag that can have a certain value, in this case href sets the URL we want to open whenever someone clicks our link.

There are other ways of producing HTML, like Pug:

a(href='https://www.google.com') Click here to go to Google!

However it will all end up being HTML in the end, so I suggest sticking to plain HTML for now and trying cool stuff like Pug later on.

If you want to see a complex example of HTML try the following (instructions for Google Chrome):

  1. Press F12 - A console should appear!
  2. Select the Elements option;
  3. You should see the HTML of the current page with the corresponding CSS of the currently selected element on the right;

Don’t be scared right away if it all seems really complicated, there are many tags and rules but we’ll talk more about this later.

CSS

CSS is what makes the structure of your page look modern and pretty! It also makes your pages responsive. If you don’t know what responsive means, a website that is responsive is a website that looks and behaves perfectly well in multiple devices and their respective resolutions. It will look perfectly fine on mobile for example, adjusting everything it needs to accommodate every element in the best way.

Let’s say we had a span (which is a label/text) in our HTML and we wanted it to be blue.

HTML:

<span>This should be blue!</span>

CSS:

span {
	color: blue;
}

This will make every span in our page blue, but since we only have one in our example that’s fine!

Like HTML and Pug, there are lots of other cool ways of doing CSS, like Sass and Less for example. However, I also suggest sticking with plain CSS for now and trying this cool stuff later on.

More importantly for now, there are lots of CSS frameworks that will help us build a nice responsive website without much hassle. Unlike the previous point, I highly suggest picking a CSS framework for Web Development. One of the most popular CSS frameworks is Bootstrap but there are many others that are worth a look. We’ll pick a cool CSS framework for our project later!

JS

So, if HTML gives us the structure of our page and CSS makes it look pretty, what does JS do?
JavaScript, not to be confused with Java (they’re totally different languages!), gives functionality and interactivity to our pages. This means everything from cool complex animations to making requests to the Back-end. It will seem a bit daunting at first but JS has evolved quite a lot and, in my honest opinion, is the best language to learn in 2018, since it will allow you to do pretty much anything. As someone used to strongly typed languages like C#, Java and so on, JS felt very loose at the beginning but I quickly got the hang of it.

JS is available on your browser right away, so let’s do some tricks to try it out. First, we need to open the browser’s console, which in most browsers will open when you press F12 on your keyboard. If the console screen is not selected, make sure you select it.
With the console open, you should see a white screen. Try typing 1+1 for example and pressing enter to see the result - it should return 2 right below your instruction. If you press up on your keyboard, the console should show your previously inserted instruction.

There are two key concepts that we must learn if we're new to programming in general: variables and functions. A variable is a space in memory where we declare a name and store a value. A function is an instruction that optionally receives some values as parameters and optionally returns a value. In JS, functions that do not explicitly return a value always return undefined which is a special value of JS for something that is not defined, that is, its value was never assigned. I'll explain more about these concepts in the next articles.

For now let's try something like:

console.log('Hello World!');

In this case the JS function console.log tells the browser to show on the console whatever is written inside its parenthesis. Like many other programming languages, ; signifies the end of an instruction. ' or " are equivalent in JS and both signify that whatever’s inside them should be considered a string, which is nothing more than a literal text.

Let’s try something a bit more complex:

var name = prompt('What is your name?');
alert('Hi ' + name + '!');

If you try it out the browser should ask your name and then greet you right after!

Let’s try to understand the code, one thing at a time.
The very first thing the code does is declaring a variable called name. A variable is a space in memory that will hold a value, for example:

var myname = 'Nuno';

This means that myname will have the value ‘Nuno’. If you try it on the console, whenever you enter myname on the console it will return the text ‘Nuno’.

On the more complex code above, we’re saying we want name to be the result of the JS function prompt. The function prompt will show the user whatever text is inside its parenthesis and return the value that the user enters in the box. This means that at the end of this line, name will hold the value that the user typed on the box that was shown.

After that, we call the JS function alert. This function shows a box with whatever text is inside its parenthesis. Here, we concatenated our text, which means, we mixed some text inside ' with text that we know is inside the variable name.

Like HTML and CSS, you can also use cool stuff like TypeScript to produce JS. However, I suggest sticking to plain JS for now and trying TypeScript later on.
