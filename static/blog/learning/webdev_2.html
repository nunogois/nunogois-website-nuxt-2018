Node.js

This is a very interesting one. Remember how we talked about JavaScript and how it was part of the big Triforce of the Front-end, giving functionality to our pages? Well, thanks to Chrome’s V8 JavaScript Engine, with Node it is also possible to run JavaScript on the Back-end! This means that with Node you only need to know JavaScript to develop both Front-end and Back-end.

In this case, we’ll use Node to develop a very simple Back-end that will simply serve our Front-end files.

We’ll need npm available on the terminal to proceed. What is npm? npm is a JavaScript Package Manager. There are tons of packages available and npm helps us manage those packages.
You can also try Yarn in the future which does pretty much the same thing with some differences but since npm is included with Node, that’s what we’ll be using!

Project Structure

Let’s start by creating our first file: package.json - this file is a configuration file that will contain information about our website.

JSON

JSON is a big part of JavaScript and is pretty easy to learn. When you think about simple variables, they usually hold values like so:

var name = 'Nuno';
var age = 28;

That’s cool and all, but with JSON we’re able to do this:

var me = { "name": "Nuno", "age": 28 };

Woah, what happened? You can go ahead and try it out on your browser’s console. When you enter me as an instruction afterwards, it returns our JSON. If you enter me.name it will return ‘Nuno’ and if you enter me.age it will return 28. JSON made it possible to define a complex object with several values.
What if we wanted to update the property name of our variable me? We can simply do something like me.name = 'Góis'. Want to add a new property? That’s easy too: me.job = 'Full-Stack Developer'. Cool, right?

What you need to remember is that JSON standard syntax is like this:

{ "key": "value", "key2": "value2", "key3": "value3" }

Back-end

As you might have guessed from the file extension, the package.json we created will hold a JSON with all the information needed to run our Back-end.
In this case, let’s input something like this in our package.json:

{
	"name": "hello_world",
	"version": "0.0.1",
	"scripts": {
		"start": "node server.js"
	}
}

This tells that our project’s name is hello_world and its version is 0.0.1, you can change these however you want. See scripts? It’s also defined as a JSON so that people can define several different scripts. In this case we define the script start - here we’re saying that whenever we run npm run start, we should in fact run node server.js, which is what will run our server.js file and start up our server.
The file server.js doesn’t exist yet, so let’s save the current package.json file (I like using CTRL+S for this) and create our new file: server.js - which will contain our Back-end logic.

Before adding our server.js code however, we need to install a package that our code will use that will help us run our Back-end. I like using express for this, so let’s install it. One of VS Code cool features is that it includes a terminal right on the editor. If you’re not seeing it, simply go to View > Integrated Terminal or drag up from the blue bar on the bottom. Select Terminal if it isn’t selected yet and run the command npm install express - This will install express on our project and add this dependency to our package.json automatically.
After it successfully installs, add this code to our server.js:

var express = require('express');
var app = express(); 

app.use(express.static('public')); 

app.listen(3000);

That’s it for now! 😃
Our first two lines declare the variables express and app. The variable express will be our required express package that we added earlier and app will build/initialize a new express instance.
Next, we tell app that it should use a folder called public in our project to serve our Front-end files. This folder can be whatever name you choose to call it, some people call it app or static, for example. I’ll call it public for now.
The last line will tell our app to start listening on port 3000!

Front-end

We’re almost there, now let’s add our public folder to our project to hold our Front-end files.
Inside, we’ll create an index.html file to hold our HTML. On the Web, the name index is commonly given to default pages.

The basic structure of a modern HTML file goes something like this:

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title>Page Title</title>
	</head>
	<body>
	</body>
</html>

The <!DOCTYPE html> tag tells that our document is HTML5 and the <html> tags simply tell where our HTML starts and ends.
The <head> tag holds information about our HTML page, like the type of characters we will use and our page title, in this case.
The <body> tag holds the content of our page, whatever we want to display on our page!

Let’s change our page title and body to something a bit more appropriate. We’ll put the following HTML in our index.html:

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title>Hello World!</title>
	</head>
	<body>
		Hello World! This is my first website.
		<br>
		If you want to make a cool website like this, you can follow <a href="https://www.nunogois.com/blog">Nuno Góis Learning Series</a> as well!
	</body>
</html>

I already talked about the <a> tag in the previous article, the only new tag here is the <br> tag which means a line break in our text.

Let’s see what this looks like!
Let’s save everything we edited so far. In VS Code terminal, let’s run npm run start. Remember what this does? It should run server.js and start our project on port 3000!
Unless you face an error, try going to http://localhost:3000 on your browser and see what happens!
Hopefully you’ll see the result of our HTML. Despite what we said in the page, it isn’t that cool yet, is it? Let’s change that soon. But hey, it’s a start! Why localhost:3000? Well, localhost is a special keyword that will always point to your own computer and :3000 tells what port we’re accessing, in this case 3000 which is what we set in server.js.

If you want to stop running your server, simply press CTRL+C on the terminal. You can start it again anytime by running npm run start again.

Now let’s spice things up a bit!

It’s possible to have JS and CSS inside our HTML file, but I believe having everything on their own files is a good practice. Because of that, let’s add index.js and index.css files to our public folder.

We still need to tell our HTML that we want it to use these files for our page, so we add them on our <head> tag, like this:

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title>Hello World!</title>
		<link rel="stylesheet" href="index.css"> <!-- NEW! -->
		<script src="index.js"></script> <!-- NEW! -->
	</head>
	<body>
		Hello World! This is my first website.
		<br>
		If you want to make a cool website like this, you can follow <a  href="https://www.nunogois.com/blog">Nuno Góis Learning Series</a> as well!
	</body>
</html>

See the <!-- NEW! --> I added to the code? Those are just HTML comments. Comments don’t matter in terms of code or functionality, they’re only there for informative purposes, in this case to show you where I added the new tags! You can go ahead and delete them if you want.
Other languages have comments like these // This is a comment! or these /* This is also a comment! */, for example.

Now that we added our CSS and JS files, let’s test them to see if they’re working.

First, let’s use our CSS file to align our text to the center of the page:

body {
	text-align: center;
}

Here we’re saying that the text inside our body element should be aligned to the center.

Now, let’s use our JS file to show something on the console, just to see if it’s working:

console.log("Yep, it's working!");

Go ahead and run your server again or reload the http://localhost:3000 page. Check if your text is now centered and that the text Yep, it's working! was shown on the console.

Great, now we can experiment all we want with our new files and check out what happens!
We still can’t show off to anyone, unless you know how to open ports on your router and/or firewall - in which case you can open port 3000.

JS (index.js):

// If someone presses the Enter key on the name field we should greet them!
document.getElementById('name').addEventListener("keyup", function(event) { // Whenever someone presses a key on this field
	if (event.keyCode === 13) { // If the key that was pressed is the Enter key
		greet(); // Call the function
	}
});

// Function greet:
function greet() {
	var name = document.getElementById('name').value; // Get the current value of the name field

	// Show noty while setting some options:
	new Noty({
		text: 'Hello ' + name + ', nice to meet you!',
		layout: 'topCenter',
		theme: 'nest',
		type: 'info',
		animation: {
			open: 'animated fadeInDown', // Animate.css class names
			close: 'animated fadeOutUp'  // Animate.css class names
		}
	}).show();
}
